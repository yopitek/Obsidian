## 5. Lessons Learned

### ✅ 做對的事
1. **使用 --ref 參考圖**: 讓 AI 參考產品實際外觀，生成的圖片更像真實產品
2. **使用 --sessionId**: 保持多張圖片風格一致
3. **指定精確顏色代碼**: #050B18, #00D4FF 等，不依賴 "dark blue" 這類模糊描述
4. **Gemini 3.0 Flash**: 比 Pro 模型生成更快
5. **簡化 prompt**: 移除過多修飾詞，專注重點

### ❌ 問題與錯誤
1. **最初沒使用 --ref**: 前幾張圖片沒有參考產品圖，導致產品外觀不像 AWUS036AXML
2. **模型名稱錯誤**: 顯示 "AWUS036AX" 而非正確的 "AWUS036AXML"
3. **超時問題**: 使用 --ref 時每次上傳圖片耗時 30s+，導致超時
4. **prompt 太長**: 某些 slide 超時因為 prompt 太複雜

### 🔧 改進建議

#### 提高畫質
1. **使用 gemini-3.0-pro**: 比 flash 更高畫質 (速度較慢)
2. **單獨生成**: 不要一次生成多張，每次一張確保成功
3. **預先上傳參考圖**: 可考慮修改腳本緩存參考圖
4. **分離 spec 和視覺**: 先用簡單 prompt 生成視覺，再手動添加文字

#### 更好的 workflow
```bash
# 1. 首先測試單張
/opt/homebrew/bin/bun /路徑/main.ts \
  --prompt "簡單描述" \
  --ref 參考圖.png \
  --image output.png \
  --model gemini-3.0-flash

# 2. 確認成功後再批量

# 3. 使用 sessionId 保持風格
--sessionId alfa-awus-001
```

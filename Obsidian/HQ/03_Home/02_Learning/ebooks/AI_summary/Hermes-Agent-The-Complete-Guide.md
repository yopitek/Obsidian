# Hermes Agent：完整指南摘要

## 一句話總結

這本書在講的不是又一個 AI Agent，而是把「如何馴化 AI、讓它越用越懂你」這件事產品化：記憶、技能、工具、反思、跨平台，全都內建成一套系統。

## 重點摘要

1. Hermes Agent 的定位，是把 Harness Engineering 從手工方法，變成可直接使用的產品。
2. 書中最核心的機制是 **Learning Loop**：Agent 每完成一次任務，就回顧做得好不好，逐步修正自己的工作方式。
3. 第二個核心是 **Three-Layer Memory**：session memory 記住當前對話，persistent memory 記住長期偏好，skill memory 則把能力流程固化下來。
4. **Skill System** 代表能力可封裝、可演化、可重複使用，Agent 不只是回答問題，而是逐步學會做一類事情。
5. **40+ 原生工具 + MCP** 讓 Hermes 可以連 GitHub、資料庫與外部服務，把能力擴展到真實工作場景。
6. **Multi-Platform Gateway** 讓同一個 Agent 出現在 Telegram、Discord、Slack、CLI 等多個入口，成為真正隨處可用的個人系統。

## 重要概念

### 1. Harness Engineering 產品化

過去你要自己寫規則、配工具、建記憶、做回饋；Hermes 把這些層全部包成內建能力，降低一般人打造個人 Agent 的門檻。

### 2. Learning Loop

這是整本書最有價值的概念：不是讓 AI 一次答對，而是讓它在任務後自我回顧，把錯誤變成未來的能力增強。

### 3. Three-Layer Memory

- **Session memory**：記住眼前任務。
- **Persistent memory**：記住你是誰、偏好什麼、常做什麼。
- **Skill memory**：把反覆有效的方法，變成固定能力。

這三層結合後，Agent 才會從「金魚腦」變成「真的越用越熟」。

### 4. Skill System

Skill 是能力單位，也是流程單位。好的 Skill 不是知識片段，而是可直接執行的做事方法。

### 5. MCP 是 Agent 的 USB 介面

只要外部服務支援 MCP，Hermes 就能接上去。這使 Agent 從聊天框走進真實工作系統。

## 日常生活如何運用

1. 把你的常見需求做成 Skill，例如每日摘要、閱讀筆記整理、行程回顧、待辦拆解。
2. 讓 Agent 記住你的偏好，例如語氣、輸出格式、資料夾規則，減少每天重新解釋。
3. 對重複犯錯的 AI，不要只抱怨，應該把錯誤轉成規則與回顧機制，這就是 Learning Loop 的精神。
4. 若你平常在手機、桌機、聊天工具之間切換，跨平台 Agent 會比單一桌面工具更接近日常使用情境。

## 工作上如何運用

1. 用 Hermes 建立團隊的內部工作 Agent，讓它熟悉專案術語、流程與常見判斷。
2. 把 onboarding、PR review、會議摘要、需求轉任務等流程做成 Skill，讓新同事和 AI 都能按同一套方式做事。
3. 透過 MCP 串 GitHub、資料庫、文件系統與通訊平台，讓 Agent 直接參與協作，而不只提供建議。
4. 用 persistent memory 保留團隊慣例與使用者偏好，讓跨 session 的工作延續性更好。
5. 在多平台入口布署同一個 Agent，可讓客服、內部助理、技術支援、內容流程共用同一套核心能力。

## 我認為這本書最值得帶走的 3 件事

1. 好的 Agent 不是回答得多聰明，而是能不能持續變得更會做事。
2. 記憶、技能、工具、回饋，四者缺一不可。
3. AI Agent 的未來不是單次對話，而是長期可演化的個人／團隊工作系統。
